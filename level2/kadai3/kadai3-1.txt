① デフォルト引数 ＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝
【概要】
引数に初期値（省略時の値）を設定できる仕組み。
関数を呼ぶときに値を渡さなければ、その初期値が使用される。

【記述方法】
〜〜関数定義〜〜
def greet(name, message="こんにちは"):
    print(message, name)
↑ messageに予め「こんにちは」が含まれている

【使い方】
greet("太郎")
結果…こんにちは 太郎
↑messageを設定してないが、設定していた「こんにちは」が入るようになっている

greet("太郎", "おはよう")
結果…おはよう 太郎

= を使って 引数名=初期値 の形で書く
デフォルト引数は後ろに書く（前に書くとエラー）


② 位置引数とキーワード引数 ＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝
【概要】
呼び出す関数に値を渡す方法のこと。
位置引数…関数定義の順番(位置)に合わせて値を渡す。
キーワード引数…引数名 = 値と指定して値を渡す。関数定義の順番とは違う順番で記述できる。

【記述方法】
〜〜関数定義〜〜
def introduce(name, age):
    print(name, age)

＊位置引数
introduce("太郎", 20)
→「太郎」「20」の順番で表示される。
　順番が入れ替わると表示順位も入れ替わる。

＊キーワード引数
introduce(age=20, name="太郎")
関数定義の順番とちがっても指定できる。

同時に使用することも可能
introduce("太郎", age=20)
※ 位置引数 → キーワード引数 の順ならOK　※逆はNG


③ 型ヒント（型アノテーション） ＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝＝
【概要】
変数や引数・戻り値の「型」を明示するため記述。
プログラム実行時の処理や動作には反映されず、エディタ上でのチェックや自分以外がコードを見る時など、
可読性・補完・バグ防止が目的で使用される。

＊引数の型ヒント
def add(a: int, b: int):
    return a + b
→ aとbどちらも整数（int）であるとわかりやすくなる

＊戻り値の型ヒント
def add(a: int, b: int) -> int:
    return a + b
→ returnでの戻り値が整数（int）であるとわかりやすくなる

＊変数の型ヒント
count: int = 10
name: str = "太郎"
→ 変数countが整数（int）で「10」、
　 変数nameが文字列（str）で「太郎」

エディタ（VSCodeなど）の補完が強くなる
mypy などの型チェックツールと相性が良い
